How the Web Works

Before we can understand how things break, we need to understand how they work. This section covers just enough HTML, CSS, and JavaScript to see where security problems come from.

⏱️ ~20 minutes
🎯 Beginner
🛠️ 1 Interactive Demo

🏗️ The Three Pillars of a Web Page

Every website you visit is built from three languages working together. As a security professional, you need to understand each one — not to build websites, but to understand how browsers process and execute code.

📄

HTML

Structure and content — headings, paragraphs, forms, links, and input fields.

🎨

CSS

Visual styling — colors, fonts, layout, and how things look on screen.

⚡

JavaScript

Behavior and logic — what happens when you click a button or type into a form.

💡

Security perspective: HTML and CSS are mostly static. JavaScript is where things get dangerous — because the browser executes it as code, not just text.

📄 HTML — Structure

HTML uses tags to mark up content. Tags open and close like this: <tag>content</tag>. A browser reads these tags and decides how to display the page.

HTML <!-- A basic HTML page structure --> <html> <head> <title>My Page</title> </head> <body> <h1>Welcome</h1> <p>Enter your name:</p> <input type="text" id="nameInput"> <button>Submit</button> <div id="output"></div> </body> </html>
🔍

Notice the id attributes. These let JavaScript find and interact with specific elements on the page. The <input> field and the <div id="output"> are where user data enters and gets displayed — this is exactly where XSS attacks happen.

⚡ JavaScript — The Critical Part

JavaScript runs inside the browser and can read user input and write content back to the page. There are two key methods for writing to the page — and choosing the wrong one creates a security vulnerability.

JavaScript // Get the value from the input field const userInput = document.getElementById('nameInput').value; // Method 1 — textContent (SAFE ✅) // Treats input as plain text. HTML tags are NOT executed. document.getElementById('output').textContent = userInput; // Method 2 — innerHTML (DANGEROUS ⚠️) // Treats input as HTML. Tags ARE parsed and scripts CAN run. document.getElementById('output').innerHTML = userInput;
Method What it does Parses HTML? Safe for user input?
textContent Displays input as plain text ❌ No ✓ SAFE
innerHTML Renders input as live HTML ✅ Yes ⚠ DANGEROUS
⚠️

This is the root of XSS. When a developer uses innerHTML with user-supplied data, the browser doesn't know the difference between the developer's HTML and the attacker's HTML. It executes both.
